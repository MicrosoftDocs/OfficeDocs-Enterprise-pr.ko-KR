# <a name="accessible-diagram---sharepoint-server-exchange-lync-office-web-apps---the-mobile-landscape"></a>액세스할 수 있는 다이어그램-SharePoint Server, Exchange, Lync, Office Web Apps-는 모바일 가로 방향

**요약:** 이 문서에는 [기술 다이어그램](http://go.microsoft.com/fwlink/?LinkID=519139&amp;amp;clcid=0x409)에서 사용할 수 있는 Microsoft Exchange 2013 플랫폼 옵션 라는 다이어그램의 액세스할 수 있는 텍스트 버전입니다.
  
이 포스터에서는 응용 프로그램 및 SharePoint Server, Lync, Exchange Server 및 Office Web Apps에 대 한 모바일 브라우저 기반 가로의 자세한 레이아웃을 표시 합니다. 다음 정보를 포함 합니다. 
  
- 응용 프로그램 및 브라우저 기반 환경에 설명 하는 섹션이 있습니다.
    
- 온-프레미스 및 클라우드 기반 모바일 배포를 묘사 하는 4 개의 토폴로지입니다.
    
## <a name="applications-and-browser-based-experiences"></a>응용 프로그램 및 브라우저 기반 환경

이 섹션에서는 장치 응용 프로그램 및 SharePoint Server 2013, Exchange Server 2013 및 Lync 2013에 대 한 브라우저 기반 환경을 선택 영역에 설명 합니다. 일부 응용 프로그램 및 브라우저 기반 환경 (전화 및 태블릿) 두 폼 요소에 대해 사용할 수 있는 하 고 온-프레미스 및 Office 365 환경 모두에서 작동 하지 모든 경험을 고려해 야 합니다. 각 응용 프로그램 및 브라우저 기반 시나리오에 대 한 자세한 지원 정보가 제공 됩니다.
  
### <a name="sharepoint"></a>SharePoint

응용 프로그램 및 브라우저 기반 환경을 다음 목록은 SharePoint Server 2013에 사용할 수 있습니다. 각 환경에 함께 제공 되는 전화/태블릿 그래픽을 포함 됩니다.
  
 **현대식 보기 (브라우저 기반)**
  
이 경험을 제공합니다.
  
- Select 사이트 서식 파일 (팀 사이트, 빈 사이트, 문서 작업 영역, 문서 센터 및 프로젝트 사이트)에 기본적으로 활성화 하는 HTML5 기반 브라우저 보기입니다.
    
- 메뉴 단추, 탐색 창, 누르고-열기 링크 및 페이지 매김 같은 향상 된 기능입니다.
    
- 표준 기능 거의 없는 구성 또는 사용자 지정 필요 합니다.
    
- SharePoint Server (2013만 해당) 또는 SharePoint Online에서 선택한 전화 모바일 브라우저에 대 한 지원 합니다.
    
 **SharePoint 뉴스 피드 응용 프로그램**
  
이 응용 프로그램을 제공합니다.
  
- 새로운 공유 환경을 다운로드할 수 있는 응용 프로그램을 통해 해당 모바일 장치에서 사용할 수 있습니다.
    
- 모든 SharePoint 웹 사이트에서 뉴스 피드를 게시할 수 있는 기능 사용자, @mentions, 문서, #hashtags, 등을 수행 합니다.
    
- SharePoint Server 2013 (Windows 8, iOS) 또는 SharePoint Online (Windows Phone, Windows 8 및 Apple iOS) 사용 가능 합니다.
    
 **장치 채널 (브라우저 기반)**
  
이 경험을 제공합니다.
  
- SharePoint 2013에서 장치 채널 기능을 서로 다른 장치를 대상 디자인 하는 서로 다른 여러 방식으로 사용 하 여 단일 게시 사이트를 렌더링 하는 기능입니다.
    
- SharePoint 2013 게시 사이트에 대해서만 사용할 수 있는 장치 채널
    
- SharePoint Online을 사용 하는 경우, 온-프레미스 설치 하 고 두 장치 채널의 합계에 대 한 특정 사이트에 구성 된 기본을 포함 하는 10 가지 장치 채널을 최대 합니다.

### <a name="onedrive-for-business"></a>비즈니스용 OneDrive

비즈니스용 OneDrive는 조직에서 직원에 대 한 SharePoint 2013 온-프레미스 및/또는 Office 365 SharePoint Online 배포의 일부분으로 사용할 수 있는 클라우드 저장소. 모바일 앱 마켓플레이스 앱 및 Office Mobile 등의 옵션을 통해 Windows Phone, Windows 8, iOS 장치 및 Android 사용할 수 있습니다. 함께 제공 되는 전화/태블릿 그래픽 사용 되 고 있는 비즈니스용 OneDrive를 보여줍니다.

- 오프 라인으로 전환 여행 특정 콘텐츠를 선택 합니다.

### <a name="office"></a>Office

Office Mobile을 사용 하 여 외부 사용자가 Windows 8 태블릿, Ipad, 전화 등의 모바일 장치 브라우저에서 Office 문서를 볼 수 있습니다. Office Web Apps는 Word, Excel, PowerPoint 및 OneNote 응용 프로그램에 온라인 수반 합니다. Office Web Apps 여러 장치 브라우저 및 제조업체에 사용할 수는 있지만, 여러 장치 달라질 수 있습니다 경험을 확인 합니다. 예, Windows 8 태블릿 및 iPad 편집 기능을 제공 전화 이러한 앱을 볼 수만 사용 하지만 합니다. 함께 제공 되는 그림 Microsoft PowerPoint Microsoft Excel에 대 한 iPad에서 Office Web Apps 경험을 표시 합니다. 온-프레미스와 온라인 배포에 사용할 수 있는 것입니다.
  
 **고화질 보기**입니다. 확인 하 고 하 고 장치 브라우저에서 바로 Office 문서를 공유할 하 고, 모양과 때 처음 만들 때 사용한으로 폴란드어 같은 문서를 가질 수 알고 수 합니다. 함께 제공 되는 그림 태블릿 컴퓨터에서 Office Web Apps 경험을 표시 합니다.
  
- Word Web app, 그래픽, 이미지 및 뛰어난 충실도, 보기 및 메모 추가 (영문)에 대 한 새 지원을 포함 하 여 포함 된 레이아웃을 참조 합니다.
    
- PowerPoint Web App를 사용 하 여 고해상도 슬라이드, 전체 전환 및 볼 수 있는 메모와 바탕 화면에서 마찬가지로 프레젠테이션을 볼 합니다. 재생할 수 있습니다도 다시 태블릿 또는 전화에서 바로 오디오 및 비디오 파일입니다.
    
- Excel Web app를 변경할 수 있으며 고화질 차트 3 차원, 향상 된 쿼리 테이블 및 데이터 유효성 검사 지원, 슬라이서 기능 및 피벗 차트 대화형 작업을 포함 하 여 심도 있는 통찰력에 대 한 데이터를 시각화할 수 있습니다.
    
 **브라우저 (Windows 8/iPad)에서 편집**합니다. 브라우저 기반 Office Web apps 필수 하 고 친숙 한 도구를 사용 하 여 편집을 할 수 있습니다. 함께 제공 되는 그림 태블릿 컴퓨터에서 Office Web Apps 경험을 표시 합니다.
  
- Excel Web app에서는 셀 병합, 상황에 맞는 메뉴, 자동 맞춤을 열, 수식 지원, 채우기 핸들 및 자동 합계와 같은 브라우저를 통해 필수적인 스프레드시트 기능을 가져옵니다. 브라우저에서 바로 인쇄 지정 하 고 이름바꾸기 또는 필요에 따라 시트를 추가 합니다.
    
- Word Web app 편집, 서식, 및 일반적으로 사용 되는 기능 및 바탕 화면에 알고 있는 Word에서 바로 가기를 사용 하 여 브라우저에서 바로 문서를 디자인 수 있습니다. 
    
- PowerPoint Web app 텍스트를 편집할 수도 있고 끌어서 놓기 작업을 사용 하 여 그림 이나, 차트 등의 요소를 이동할 수 있습니다. 이미지를 삽입할 수, 전환 및 애니메이션를 추가 하거나 브라우저에서 바로 미리 디자인 된 테마를 적용할 수 있습니다.

Microsoft 장치 관리에 대 한 새 제품 여러 장치 브랜드 있습니다. Windows Intune은 모바일 장치에 대 한 관리 클라우드 기반 플랫폼입니다. 수 또는 장치를 관리할 수 단독으로 하나를 사용 하 고, System Center 2012 R2 Configuration Manager와 결합 하 고, 해당 콘솔 내에서 관리 합니다. 이 공동 작업의 일부 장점은 다음과 같습니다.
  
- 정책 적용 하 고 Windows RT "," Windows 8.1 "," Windows Phone 8 "," iOS, "및" Android 비롯 한 여러 장치에서 관리
    
- 조직 ·에서 최대 100, 000 장치를 확장 합니다. 무선 장치 등록
    
- 응용 프로그램의 사용자 지정
    
- 장치 인벤토리를 수행합니다.
    
- 원격 장치 퇴직 및 장치 지우기
    
- 친숙 한 Configuration Manager 콘솔에서 관리
    
함께 제공 되는 다이어그램 장치 관리에 대 한 Windows Intune 및 System Center 2012 R2 Configuration Manager를 사용 하는 방법을 보여줍니다.
  
 **System Center 2012 R2 구성 관리자 및 Windows Intune를 사용 하는 것에 대 한 고려 사항:**
  
- System Center 2012 R2 Configuration Manager와 Windows Intune를 사용 하는 경우 관리 하 고 하나의 콘솔에서 조직에서 모든 장치를 관리할 수 있습니다.
    
- 구성 관리자를 Windows Intune에 연결 하려면 Windows Intune 구독 해야 하 고 서버 구성 관리자 환경에서 Windows Intune 커넥터 사이트 서버 역할을 배포 해야 합니다.
    
- System Center 2012 R2 Configuration Manager 관리 및 온-프레미스 Pc에 대 한 소프트웨어 배포를 제공합니다.
    
- 관리 되는 최신 장치는 Windows Intune에서 정책을 적용 하 고 소프트웨어 배포를 받습니다. 지원 되는 종류의 장치를 포함: Windows RT, Windows 8.1 Windows Phone 8, iOS, Android 합니다.
    
## <a name="on-premises-and-cloud-based-mobile-deployments"></a>온-프레미스 및 클라우드 기반 모바일 배포

토폴로지에 여기 SharePoint server, Lync, 나타내고 Exchange 이러한 플랫폼을 구현 하기 위한 기본 설정된 방법 설명 하지 않습니다. 단순히 토폴로지를 고유한 네트워크 요구 사항 및 보안 고려 사항에 따라 다르게 예제를 제공 합니다.
  
### <a name="sharepoint-deployment"></a>SharePoint 배포

함께 제공 된 그림에는 SharePoint Server 2013의 엑스트라넷 배포를 보여줍니다. 장치 Secure Socket Layer (SSL) 또는 VPN 게이트웨이 사용 하는 등의 옵션을 사용 하 여 SharePoint 2013 웹 프런트엔드 서버에 안전 하 게 연결 하는 것이 좋습니다.
  
SharePoint 역방향 프록시 및 장치 간의 암호화 된 통신을 위해 TCP 포트 443 (SSL)를 사용할 수 있습니다. 인터넷에서 외부 액세스에 대 한이 포트는 방화벽이 나 라우터에에서 인바운드 및 아웃 바운드 트래픽에 대 한 열 수를 해야 합니다. 
  
웹 프런트엔드 서버에서 사이트를 액세스 하는 모바일 브라우저, http 요청 IIS 하려고 합니다. Http 요청 USERAGENT 필드는 요청 하는 모바일 브라우저에 대 한 정보를 포함 합니다. 상자 현대식 보기 또는 장치 채널 기능을 사용 하는 사용자 지정 사이트 구현에 대 한 브라우저 리디렉션 결정에 대 한이 정보를 사용할 수 있습니다.
  
SharePoint 2013에 대 한 응용 프로그램은 이제 독립 실행형 제품 및 Office Web Apps 설치에서 분리 되었습니다. Office Web Apps를 포함 하 여 모바일 클라이언트 액세스 권한을 제공 하려면 설치 된 Office Web Apps 서버를 사용 하 여 SharePoint 2013을 구성 해야 합니다. 따라서 다음 SharePoint 2013 관리 셸 cmdlet을 통해 수행할 수 있습니다.
  
```
New-SPWOPIBinding -ServerName <ServerName>, where <ServerName> is the FQDN name of the URL that you set for the internal and external URLs.
```

### <a name="exchanage-deployment"></a>Exchanage 배포

또한 Exchange 자동 검색 서비스를 제공 및 Microsoft Exchange ActiveSync, Outlook Web app 등의 응용 프로그램을 호스팅하는 온-프레미스 Exchange 클라이언트 액세스 서버에 장치를 연결 합니다. 외부 장치 및 Exchange 클라이언트 액세스 서버 간에 SSL 암호화를 사용 하는 것이 좋습니다.
  
Exchange 자동 검색 및 Exchange ActiveSync와 같은 장치 트래픽과 포트 443 (HTTPS)을 통해 처리 됩니다. Direct Push 기능 Exchange 2013 유지 하는 모바일 장치를 현재 무선 또는 셀룰러 네트워크 연결을 통해 합니다. 새 콘텐츠를 동기화 할 준비가 된 경우에 모바일 장치를 알립니다. 발 조직 및 통신 회사의 모든 방화벽 업데이트로 HTTPS 요청을 지원 합니다. 기본적으로 직접 올리기 폴더에서 15 분에 대 한 모든 변경 내용에 대 한 폴링할 하 여 장치 업데이트 구성 됩니다. 기본 15 분 직접 올리기 구성 아래에 해당 하는 경우 방화벽에 시간 제한 값을 늘려야 할 수 있습니다.
  
Exchange ActiveSync 서비스를 호스트 하는 Exchange 클라이언트 액세스 서버 (CAS). 전화 또는 태블릿 및 Exchange 2013 간에 데이터를 동기화 할 수 있도록 합니다. 
  
### <a name="lync-deployment"></a>Lync 배포

일반 워크플로에 대 한 외부 모바일 장치에서 예제 외부 Lync URL을 HTTPS 가져오기 요청을 보내: LyncDiscover.contoso.com 합니다. 프런트엔드 서버 (또는 서버 풀)에 포트 443 통해 요청을 전달 하는 역방향 프록시 서버에 요청을 받습니다. 역방향 프록시 서버 lyncdiscover.contoso.com 게시 규칙을 있어야 합니다. 또한 방화벽 및 역방향 프록시 서버는 별도 컴퓨터에서 호스팅할 수는 것이 좋습니다.
  
프런트엔드 서버에서 자동 검색 서비스는 사용자의 홈 풀에 대 한 웹 서비스 Url을 검색 하 고 역방향 프록시 서버를 통해 모바일 장치에 해당 정보를 반환 합니다. 자동 검색 서비스에는 모바일 장치를 사용자가 Url을 수동으로 입력할 필요 없이 Lync Server 웹 서비스를 찾을 수 있습니다.
  
모바일 장치는 방화벽 내부에 로그온 하는 경우 HTTPS Get 명령은 LyncDiscoverInternal.com 등 Lync Front End Pool에 직접 전송 됩니다. 요청은 포트 443 모바일 장치를 다시 통신 하는 자동 검색 서비스를 사용 하는 동일한 포트에서 수신 됩니다.
  
모바일 기능을 지원 하기 위해 두 DNS 레코드를 구성 해야: LyncDiscover.contoso.com 및 LyncDiscoverInternal.com에 대 한 하나입니다. 외부 FQDN; 방화벽 외부에서 액세스할 수 있어야 합니다. 내부 FQDN만 방화벽 내에서 액세스할 수 있어야 합니다.
