# <a name="use-the-page-diagnostics-tool-for-sharepoint-online"></a>SharePoint Online 용 페이지 진단 도구 사용

이 문서에서는 **SharePoint Online**에서 권장 되는 방법의 하위 집합에 대해 페이지 진단 도구를 사용 하 여 클래식 팀 사이트의 클래식 게시 페이지 및 페이지를 분석 하는 방법에 대해 설명 합니다. 
  
게시를 사용 하도록 설정 하지 않은 팀 사이트는 CDNs를 사용할 수 없지만 나머지 규칙은 모두 적용 됩니다. 게시 하면 추가 오버 헤드가 추가 되므로 게시를 켜지 않아 CDN 기능을 사용 하도록 설정 하면 페이지 로드 시간에 부정적인 영향을 줄 수 있습니다.

**V 1.05가 릴리스 되었으며, 이미 설치 되어 있는 경우 내선 번호를 업데이트 하세요**. 어떤 버전을 갖고 있는지 확실 하지 않은 경우에는 "정보" 링크를 클릭 하 여 확인 하십시오.
  
> [!IMPORTANT]
> 이 도구는 SharePoint 사이트 페이지를 검토 하도록 설계 되었으므로 문서 라이브러리 또는 시스템 페이지에 대해서는 페이지 진단 도구가 실행 되지 않습니다. *Allitems.aspx* 페이지는 시스템 페이지입니다. 시스템 페이지에서 도구를 실행 하려고 하면 "이 응용 프로그램은 SharePoint 페이지 에서만 실행 해야 합니다." 라는 메시지가 표시 됩니다. <br/> ![SharePoint 페이지에서 실행 해야 합니다.](media/34aadfff-1009-496b-9c87-4fc2780e017c.png)<br/>라이브러리 또는 시스템 페이지를 평가 하는 데 값이 없으므로 도구에서 오류가 발생 하지 않습니다. 이 도구를 사용 하려면 비시스템 SharePoint 페이지로 이동 하세요. SharePoint 페이지에서이 문제가 발생 하는 경우 고객이 보았을 때 MasterPage를 확인 하 여 SharePoint MetaTags를 제거 하 고 페이지를 더 이상 SharePoint 페이지로 설정 하지 마십시오. 도구에 대 한 의견을 제공 하 시겠습니까? 정보 탭을 클릭 하 고 [의견 제공 링크](https://go.microsoft.com/fwlink/?linkid=874109)를 따라 이동 합니다. 
  
## <a name="install-the-page-diagnostic-tool"></a>페이지 진단 도구 설치

> [!IMPORTANT]
> Microsoft는 사용자가 방문 하는 데이터 나 웹 사이트를 읽지 않으며,이 도구를 사용 하 여 개인 정보, 웹 사이트를 캡처하거나, 정보를 다운로드 하지 않습니다. 도구에서 기록 되는 유일한 정보는 테 넌 트 이름, 규칙 수 및 도구를 실행할 때 지원 로깅 옵션을 사용 했는지 여부입니다. 이 정보는 Microsoft가 고객에 게 발생 하는 문제를 분석 하 고 지원 로깅 기능이 오용 되 고 있지 않은지 확인 하기 위한 것입니다.

1. Chrome 브라우저를 사용 하 여 [도구에 대 한 링크](https://chrome.google.com/webstore/detail/inahogkhlkbkjkkaleonemeijihmfagi) 를 직접 열거나 [Chrome Browser WebStore](https://chrome.google.com/webstore/search/page%20diagnostics%20for%20sharepoint) 에서 검색을 열고 브라우저 확장을 설치 합니다. 스토어의 설명 페이지에서 제공 되는 사용자 개인 정보 보호 정책을 확인 하세요. 브라우저에 도구를 추가할 때 다음과 같은 사용 권한 표시가 나타납니다.<br/>![Chrome 스토어 사용 권한](media/e9fbcef0-1171-43ac-8ea8-c2b5be1b7925.png)<br/>   페이지의 웹 파트 및 사용자 지정 내용에 따라 페이지에 SharePoint 외부 위치의 콘텐츠가 포함 될 수 있기 때문에이 알림 메시지가 표시 됩니다. 즉, 도구는 시작 단추를 클릭 하 고 도구를 실행 하는 활성 SharePoint 탭에 대해서만 요청 및 응답을 읽습니다. 이 정보는 웹 브라우저에 의해 로컬로 캡처되고 도구에서 JSON으로 내보내기 링크를 통해 사용할 수 있습니다. **이 정보는 Microsoft에서 보내거나 캡처할 수 없습니다.** (이 도구는 [여기](https://go.microsoft.com/fwlink/p/?linkid=857875)에서 액세스할 수 있는 Microsoft 개인 정보 취급 방침을 고려 합니다.)<br/><br/>도구에서 "JSON으로 내보내기" 기능을 사용 하는 경우에는 "다운로드 관리" 권한이 필요한 이유가 있습니다. 결과에 Url이 포함 되 고 PII (개인 식별이 가능한 정보)로 분류 될 수 있으므로 조직 외부의 JSON 파일을 공유 하기 전에 회사에서 제공 하는 개인 정보 보호 지침을 따르세요.
    
2. (선택 사항) Chrome incognito 모드에서이 도구를 사용 하려면 내선 번호로 이동한 후 **incognito에서 허용**을 클릭 합니다.
    
3. 검토할 SharePoint Online의 SharePoint 클래식 게시 페이지로 이동 합니다. 페이지에 있는 항목의 "지연 로드"가 허용 됩니다. 따라서 **이 도구는 자동으로 중지 되지**않습니다. 중지를 클릭 하 여 수집을 중지할 수 있습니다 ****. (이는 모든 페이지 로드 시나리오에 맞게 설계 되었습니다.) **중지**를 클릭 하기 전에 네트워크 추적 데이터가 완료 되었는지 확인 합니다. 그렇지 않으면 부분 추적을 갖게 됩니다. 또한이 도구는 브라우저 확장 이며 여러 탭 이나 창을 열면 한 번에 하나의 활성 인스턴스만 실행할 수 있습니다. 이는 브라우저의 확장에 대 한 제한 사항입니다. 
  
4. 내선 번호 로고 클릭 ![SharePoint 로고에 대 한 페이지 진단](media/60a3e44d-1b59-483f-b50f-d580044d921a.png) 도구를 로드 하려면 다음과 같은 확장 팝업 창이 표시 됩니다.<br/> ![페이지 진단 도구 팝업](media/b01fa00e-c5f3-4c37-91f2-6edd096cf87e.png)<br/>시작 및 중지 작업 시작을 클릭 하면 기본 개념을 따르면 페이지가 다시 로드 되 고 수집이 시작 됩니다.

이 도구에서 제공 하는 정보에 대 한 자세한 내용을 보려면 다음 섹션을 읽어 보십시오.

## <a name="what-youll-see-in-the-page-diagnostics-tool"></a>페이지 진단 도구에 표시 되는 항목
    
1. **정보** 링크는이 문서에 대 한 링크를 포함 하 여 도구에 대 한 일반적인 지침과 세부 정보를 제공 합니다. 또한 SharePoint 성능 권장 사항에 대 한 직접 링크, 타사 공지 및 도구에 대 한 의견을 제공 하는 옵션이 포함 되어 있습니다. 
    
2. **상관 관계 ID, SPRequestDuration, SPIISLatency**, **페이지 로드 시간**및 **URL** 세부 정보는 정보 제공 용으로, 몇 가지 용도로 사용할 수 있습니다. 
    
  - Microsoft 지원 팀과 협력 하 여 추가 진단 데이터를 가져올 수 있도록 하는 경우 **CorrelationID** 가 중요 한 요소입니다. 
    
  - **Sprequestduration** 은 페이지를 처리 하는 데 걸리는 시간입니다. 이 시간을 초과 하는 경우에는 서버가 정상적으로 수행 되 고 있지 않다는 것을 의미 하는 것은 아니지만, 구조적 탐색, 큰 이미지, 다양 한 API 호출 등이 모두 서버 시간에 영향을 주는 경우와 같이 서버에 대 한 호출 수와 로드를 반영할 수 있습니다. . 
    
  - **SPIISLatency** 는 웹 프런트 엔드 서버에서 페이지 로드 요청을 받았을 때 수행 되는 시간 (밀리초)입니다. 페이지 처리를 시작할 대기 시간을 나타내는 표시기 이며 웹 응용 프로그램이 응답 하는 데 걸리는 시간이 포함 되지 않습니다. 
    
  - **페이지 로드 시간은** 요청이 요청을 받은 시간부터 브라우저가 응답을 받았으며 읽은 시간까지 페이지에 기록 되는 시간입니다. 컴퓨터 성능 및 브라우저를 로드 하는 데 걸리는 시간에 따라 모든 추가 시간이 영향을 받습니다. 
    
  - **URL** (Uniform resource Locator)은 현재 페이지의 웹 주소입니다. 
    
3. [ **진단** 탭](#how-to-use-the-diagnostic-tab) 에 규칙이 나열 되 고 규칙 중 하나라도 빨간색 ![십자형](media/9859ac84-be43-4eae-984c-e0e827f5a228.png)으로 표시 되어 있으면 페이지에서 문제가 확인 된 것입니다.<br/>각 규칙에는 항목이 빨간색 인 경우 클릭 하는 "추가 정보" 링크가 있습니다. 그러면 해당 규칙의 세부 정보로 이동 하 여 문제를 수정 하는 방법을 안내 합니다.<br/>![진단 Red-규칙 열기](media/1598f0f7-3103-4613-8787-dfec6fffd40a.png)

4. [ **네트워크 추적** 탭](#how-to-use-the-network-trace-tab) 에는 페이지 빌드 요청 및 응답에 대 한 세부 정보가 제공 됩니다.

## <a name="how-to-use-the-diagnostic-tab"></a>진단 탭을 사용 하는 방법

1. **표준 사용자로 실행 확인**  서비스 계정, 관리자 또는 사이트 모음 관리자 또는 상승 된 권한을 가진 계정으로 로그인 한 경우 페이지 성능 확인이 수행 되어서는 안 됩니다. 이러한 유형의 계정에 대 한 추가 스크립트 및 기능은 특별히 로드 되므로 페이지 성능이 실제로 표현 되지 않습니다.
    
2. **SharePoint에 대 한 요청 확인**  서버에 대 한 데이터 및 요청 양은 오버 로드 된 페이지로 인해 성능이 저하 될 수 있는 것으로 제한 됩니다. 이 검사는 SharePoint에 대 한 요청 수를 확인 하 고 요청이 6 회 요청을 넘을 때이를 권고 합니다. 대부분의 요청은 캐시 되어야 하며, 따라서 모든 페이지 로드에 대해 호출 되지 않습니다. 각 사용자 및 모든 사용자별로 페이지에 대 한 호출의 양을 줄이기 위해서는 캐시를 설정 하 고 최소 15 분 동안 사용 해야 합니다. 일반적으로이 문제는 대부분의 경우 데이터를 매일 변경 하지만 페이지에서 각 사용자의 각 페이지에 대 한 데이터를 확인 하 고 자주 사용 하지 않는 경우가 많습니다.
    
3. **CDNs 사용 확인**  Microsoft는 CDNs (콘텐츠 배달 네트워크)를 제공 했으며 여기에서 언급 한 것은 SharePoint Online 콘텐츠 배달 네트워크입니다. 여러 유형의 사용 가능한 형식과 SharePoint CDNs와 같은 여러 CDN 서비스와 Azure의 CDNs가 있습니다. [다음 지침을 따르십시오](https://go.microsoft.com/fwlink/?linkid=873250).
    
4. **큰 이미지 크기 확인**  PNG와 같은 향상 된 웹 형식을 활용 하 여 웹에 맞게 이미지를 최적화 해야 합니다. 이미지 변환도 활용 해야 하며 SharePoint에서 직접 사용할 수 있습니다. = 100kb 보다 큰 이미지/이미지 변환은 웹에 최적화 되지 않은 것으로 강조 표시 됩니다. [다음 지침을 사용 하 여 이미지를 최적화](https://go.microsoft.com/fwlink/?linkid=873251)합니다.
    
5. **구조적 탐색 확인**  구조적 탐색은 원래 SharePoint 온-프레미스에서 사용 하도록 디자인 되었으며 개체 캐시도 사용할 수 있습니다. 구조적 탐색은 SharePoint Online에서 사용 하지 않는 것이 좋지만 관리 되는 탐색 또는 사용자 지정 공급자로 변경 해야 합니다. [다음 지침을 사용 하 여 탐색을 최적화 합니다.](https://go.microsoft.com/fwlink/?linkid=873247)
    
6. **CBQ WebPart 확인** (CBQ-Content by 쿼리 WebPart)  Content by 쿼리 WebPart는 각 사용자에 대해 모든 페이지 부하에 대해 쿼리의 모든 항목을 트래버스하는 동안 높은 SQL 부하를 생성 합니다. 온-프레미스 설치와 달리이 WebPart를 채우는 데 필요한 쿼리 수를 제한 하는 데 사용할 수 있는 캐시는 없습니다. 이러한 CBQ은 느리게 수행 되며 전체 페이지 성능에 영향을 줄 수 있으므로 사용 하지 않는 것이 좋습니다. 콘텐츠 쿼리 웹 파트를 교체 하려면 CSWP (콘텐츠 검색 웹 파트)를 사용 하세요. [콘텐츠 검색 웹 파트와 관련 된 다음 지침을 사용](https://go.microsoft.com/fwlink/?linkid=873245)합니다.

## <a name="how-to-use-the-network-trace-tab"></a>네트워크 추적 탭을 사용 하는 방법
    
**Network Trace (네트워크 추적** ) 탭에는 받은 응답 뿐만 아니라 페이지를 작성 하는 요청에 대 한 자세한 정보가 제공 됩니다. 

1. **빨간색으로 플래그가 지정 된 항목 로드 시간을 찾습니다**. 각 요청 및 응답의 성능은 다음과 같이 전체 페이지 성능에 미치는 영향을 기반으로 하 여 색으로 구분 됩니다.
- 녹색: \< 500ms
- 노랑: 500-1000ms
- 빨강: \> 1000ms
<br/>![네트워크 추적](media/3cfede99-7d31-4041-888d-7bbc275cadc2.png)<br/> 위에 표시 된 이미지에서 빨간색 항목은 기본 페이지와 관련이 있습니다. 페이지가 1000ms (1 초 미만)로 로드 \< 되지 않으면 항상 빨간색으로 표시 됩니다.

2. **항목 로드 시간을 테스트**합니다. 일부 경우에는 항목이 이미 브라우저에 의해 캐시 되었기 때문에 시간 또는 색 지표가 나타나지 않을 수 있습니다. 이를 올바르게 테스트 하려면 페이지를 열고 브라우저 캐시를 지운 다음 **시작** 을 클릭 하 여 "콜드" 페이지를 로드 하 고 초기 페이지 로드를 true로 반사 해야 합니다. 이 경우 페이지에 캐시 되는 항목을 확인 하는 데 도움이 되는 것 처럼 "웜" 페이지 로드와 비교 해야 합니다. 
    
3. **문제를 조사 하는 데 도움이 되는 다른 사용자와 관련 세부 정보를 공유**합니다. 도구에서 제공 하는 세부 정보 또는 정보를 개발자나 기술 지원 담당자와 공유 하려면 위의 이미지에 표시 된 **JSON으로 내보내기를** 클릭 합니다. 이를 통해 결과를 다운로드할 수 있으며,이를 통해 JSON 파일 뷰어가 표시 됩니다.

> [!IMPORTANT]
> 이러한 결과에는 Url이 포함 되며 PII (개인 식별이 가능한 정보)로 분류 될 수 있습니다. 해당 정보를 배포 하기 전에 조직의 지침을 따라야 합니다. 

## <a name="engaging-with-microsoft-support"></a>Microsoft 지원 제공
   
성능에 대 한 지원 사례를 직접 작업 하는 경우에만 사용 해야 하는 **Microsoft 지원 수준 기능이** 포함 되어 있습니다. 이 기능을 사용 하면 지원 팀 없이 사용할 때 도움이 되지 않습니다. 실제로 페이지가 더 느리게 실행 되 고 기능을 계속 사용 하는 것이 서비스의 "오용"으로 간주 될 수 있습니다. 추가 정보가 서비스의 로깅에 추가 될 때 도구에서이 기능을 사용 하는 경우 추가 정보가 제공 되지 않습니다. 

사용 하도록 설정 하 고 페이지 성능이 사용 되는 동안 성능 2-3 저하가 발생 하는 경우를 제외 하 고는 사용자가 사용할 수 있다는 알림을 받는 것이 아니라 변경 내용이 표시 되지 않습니다. 특정 페이지와 해당 활성 세션에만 해당 됩니다. 따라서 지원 팀과 적극적으로 협력 하는 경우에만이 방법을 사용 해야 합니다.

### <a name="to-enable-the-microsoft-support-level-feature"></a>Microsoft 지원 수준 기능을 사용 하도록 설정 하려면

1. 페이지 진단 도구를 엽니다.
2. 키보드에서 ALT + e-L을 누릅니다. 그러면 **지원 수준 로깅이**표시 됩니다. 
3. 확인란을 선택한 다음 **시작** 을 클릭 하 여 페이지를 다시 로드 하 고 분석을 지원 하기 위해 자세한 정보 표시 로깅을 생성 합니다.<br/>![지원 옵션 사용](media/ddef47de-8593-4b28-9346-eb48ebf6cdab.png)
  
이에 대 한 중요 한 요소는 지원 팀이 해당 번호를 이용 하 여 필요한 정보를 추출 하는 것과 같습니다. CorrelationID (페이지 진단 도구 맨 위에)를 복사 하 고 전체 ID 없이 필요한 작업을 수행할 수 없는 경우에도이 기능을 지원 하도록 지정 하세요.
