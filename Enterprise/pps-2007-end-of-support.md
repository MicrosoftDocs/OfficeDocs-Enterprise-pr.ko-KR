# <a name="performancepoint-server-2007-end-of-support-roadmap"></a>PerformancePoint Server 2007 지원 종료 로드맵

Office 2007 서버 및 응용 프로그램 서버 및 비즈니스 인텔리전스 (BI) 솔루션의 일부로 사용 하 여 수 있는 응용 프로그램을 포함 하 여 지원의 끝에 도달 했습니다. 다음 표에 영향을 받는 BI 응용 프로그램:
  
|**Microsoft BI 응용 프로그램**|**종료 된 날짜 지원**|
|:-----|:-----|
|ProClarity Analytics Server 6.3 서비스 팩 3  <br/> ProClarity 데스크톱 Professional 6.3  <br/> ProClarity SharePoint 뷰어 6.3  <br/> |2017년 7월 11일  <br/> |
|SharePoint Server 2007 서비스 팩 3  <br/> |2017년 10월 10일  <br/> |
|PerformancePoint Server 2007 서비스 팩 3  <br/> |2018년 1월 9일  <br/> |
   
자세한 내용은 [2007 서버와 클라이언트가 Office에서 업그레이드 하는데 도움이 되는 리소스](upgrade-from-office-2007-servers-and-products.md)를 참조 하십시오.
  
## <a name="what-does-end-of-support-mean"></a>지원 의미의 역할 끝 무엇입니까?

예: PerformancePoint Server 2007 s p 3, ProClarity 소프트웨어 및 SharePoint Server 2007 s p 3은 모든 Microsoft 제품에는 Microsoft의 새로운 기능, 버그 수정 및 보안 업데이트를 제공 하는 지원 주기를 했습니다. 일반적으로 제품의 초기 릴리스 날짜 로부터 10 년 동안 지속 되는 제품에 대 한 수명 주기 및 해당 주기 끝이 제품의 지원 종료 라고 합니다. ProClarity, PerformancePoint Server 및 SharePoint Server 2007 지원의 끝에 도달 했습니다,으로 Microsoft 제공 하지 않습니다.
  
- 발생할 수 있는 문제에 대 한 기술 지원
    
- 버그 수정 하는 검색 되 고 안정성과 사용 가능성 서버 영향을 줄 수 있는 문제에 대 한
    
- 검색 되는 하 고 서버 또는 응용 프로그램을 보안 위반에 취약 해질 수 있는 취약점에 대 한 보안 수정
    
- 표준 시간대 업데이트
    
ProClarity, SharePoint Server 2007 s p 3 및 PerformancePoint Server 2007 s p 3의 설치 계속 지원 끝난 경우에 실행 됩니다. 그러나 마이그레이션하는 이러한 응용 프로그램에서 가능한 한 빨리 것이 좋습니다.

### <a name="what-if-i-need-help-selecting-a-solution"></a>솔루션을 선택 하는 지원이 필요한 경우에 어떻게

다양 해 진 너무 많은 BI 옵션을 사용할 수 있는, 최상의 옵션을 결정 하는 너무 복잡 보일 수 있습니다. 하는데 도움이 되는 온라인 가이드를 제공 됩니다. [분석 및 보고를 위한 선택 Microsoft BI (비즈니스 인텔리전스) 도구](https://go.microsoft.com/fwlink/?linkid=839877)를 참조 하십시오.
  
### <a name="what-happens-if-i-dont-upgrade-now"></a>지금 업그레이드 하지 않으면 어떻게 됩니까?

지금은 업그레이드 하지 않도록 선택할 수 있습니다. 기존 서버 및 응용 프로그램은 계속 실행 합니다. 그러나 지원 끝난 후에 보안 업데이트-를 포함 하 여 더 이상의 업데이트-받을 수 없습니다. 및 서버 응용 프로그램 문제가 발생할 경우 도움말을 보려면 다음 Microsoft 기술 지원 서비스에서 볼 수 없습니다.

다음 표에서 Office 365와 함께 Excel을 사용 하 여 SharePoint Server와 함께 Excel을 사용 하 여 아래 추가 정보로의 주요 장점 비교 합니다.

#### <a name="excel-with-office-365-in-the-cloud"></a>Office 365 (클라우드에 있음)를 사용 하 여 Excel

Office 365로 이동 하는 경우 최신 서비스 및 응용 프로그램을 Excel 2016 및 Excel 온라인 포함 해야 합니다. PerformancePoint Services 아니므로 Office 365에서 사용할 수 있는 Excel 통합 문서 또는 다른 보고서를 사용 하 여 콘텐츠 PerformancePoint 대시보드를 교체 합니다. Excel 2016 새 차트 종류를 많이 있으며 Excel에서 눈에 띄는 대시보드를 만드는 작업은 이전 보다 쉽게 좋은 소식이 됩니다. 및 새로운 기능을 정기적으로 추가 되 고 됩니다. 자세한 내용은 [Windows 용 Excel 2016의 새로운 기능](https://support.office.com/article/5fdb9208-ff33-45b6-9e08-1f5cdb3a6c73.aspx)을 참조 합니다.
  
및의 자 50 개 이상의 Office 365를 구입 하면 Microsoft FastTrack 팀 수 가져올을 설정 하도록 도와줍니다. 자세한 내용은, [FastTrack](https://www.microsoft.com/fasttrack/microsoft-365/office-365)을 참고 하십시오.
  
#### <a name="excel-with-sharepoint-server-on-premises"></a>SharePoint Server (온-프레미스)를 사용 하 여 Excel

SharePoint의 최신 버전으로 업그레이드 하는 경우 다음과 같이 Excel Excel 서비스 또는 Excel 온라인으로 사용할 수 있습니다.

- SharePoint Server 2013의 Excel Services
    
- Excel Online, SharePoint Server 2016에서 별도로 설치 하는 Office 온라인 서버의 일부인
    
SharePoint Server의 새 버전도의 PerformancePoint Services를 구성 하 고 Excel Services 또는 Excel Online과 함께 사용 하는 수 있습니다.
  
업그레이드 옵션 SharePoint에 대 한 자세한 내용을 보려면, [SharePoint Server 2007 끝 지원 로드맵을](sharepoint-2007-end-of-support.md)참조 합니다.
  
Excel 서비스에 대 한 자세한 내용은, [Excel Services 개요 (SharePoint Server 2010)을](https://go.microsoft.com/fwlink/?linkid=841362)참조 하십시오.
  
### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Power BI (클라우드 또는 온-프레미스)를 사용 하 여

Power BI은 데이터를 분석 하 고 정보를 공유 하는 비즈니스 분석 도구 모음입니다. Power BI 대화형 보고서 및 온-프레미스 또는 온라인 데이터 원본을 사용 하 여 대시보드를 만들 수 있습니다. 사용자를 볼 수 있으며, 보고서 및 자신의 컴퓨터 또는 모바일 장치를 사용 하 여 대시보드를 사용 하 여 수 있습니다.
  
Power BI Office 365 또는 SharePoint 서버에 포함 되지 않은 하지만 Power BI 데스크톱, Power BI 게이트웨이 및 Power BI 서비스를 포함 하는 제공 하는 별도 합니다. SharePoint Online을 사용한 power BI도 통합합니다. 시작할 수 있는 Power BI와 무료, 및 데이터 사용 및 비즈니스 요구 사항, Power BI Pro으로 업그레이드에 따라 합니다. 자세한 내용은 참조 [Power BI 란?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Reporting Services (온-프레미스)를 사용 하 여

SQL Server Reporting Services를 설치 하 고 기본 모드 또는 SharePoint 통합된 모드에서 Reporting Services를 구성 하는 기능 함께 강력한 보고 솔루션을 제공 합니다. 보고서 디자이너, 보고서 작성기 및 전원 보기를 포함 하 여 다양 한 도구를 사용 하 여 보고서를 작성할 수 있습니다. SQL Server의 최신 버전에서는 조직의 모바일 장치에 대 한 보고서를 사용 하는 기능을 제공 된 화면 크기를 확장할 수 있는 보고서를 제공 하기 위해 SQL Server Mobile 보고서 게시자를도 사용할 수 있습니다. 자세한 내용은 참조 [SQL Server Reporting Services (SSRS): 만들기, 배포 및 모바일 및 페이지가 매겨진 보고서 관리](https://go.microsoft.com/fwlink/?linkid=841342)합니다.
  
### <a name="use-performancepoint-services-on-premises"></a>PerformancePoint Services (온-프레미스)를 사용 하 여

아시다시피, PerformancePoint Server 2007은 SharePoint Server 2007에서 별도로 구입할 되었습니다. SharePoint Server 2010 이상 버전, PerformancePoint 서비스는 SharePoint 서버에서 서비스 응용 프로그램입니다. 즉, PerformancePoint 서비스를 사용 하기 위해 별도 서버 라이선스 또는 하드웨어를 구입 필요가 없습니다.
  
PerformancePoint Server 2007에서 PerformancePoint Services를 이동 하려면 최신 버전의 SharePoint Server로 이동 하 고 PerformancePoint 서비스를 구성 합니다. 버전을 이동 하는 SharePoint Server의 PerformancePoint Server 2007에서 PerformancePoint 서비스에 기존 대시보드 콘텐츠를 가져올 수 있는지 여부를 결정 합니다.
  
- SharePoint Server 2010으로 업그레이드 하는 경우에 SharePoint Server 2010의 PerformancePoint Services에 PerformancePoint Server 2007에서 PerformancePoint 대시보드 콘텐츠를 가져올 수 있습니다. 이 작동 하는 방법에 대 한 자세한 내용은 참조 [가져오기 마법사: PerformancePoint Server 2007 콘텐츠를 SharePoint Server 2010으로](https://go.microsoft.com/fwlink/?linkid=838873)합니다.
    
- SharePoint Server 2013 또는 SharePoint Server 2016를 이동 하는 경우 대부분의 경우 (예: 데이터 원본, 보고서, 성과 기록표 및 대시보드 페이지) 새 대시보드 콘텐츠를 작성할 필요 합니다.
    
PerformancePoint Services 업그레이드 계획에 시작 하려면 다음 리소스를 참조 합니다.
  
1. [SharePoint Server 2007 로드맵 지원 종료](sharepoint-2007-end-of-support.md)
    
2. 이동 하는 SharePoint의 버전을 알고 있는 경우 PerformancePoint 서비스에 대 한 해당 문서를 참조 합니다.
    
  - [PerformancePoint Services 계획(SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [SharePoint Server 2013의 PerformancePoint Services 개요](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [SharePoint Server 2016의 PerformancePoint Services 개요](https://go.microsoft.com/fwlink/?linkid=874704)
    
PerformancePoint Services로 업그레이드할 때 즐길 수 몇가지 새로운 기능 및 향상 된 기능입니다. PerformancePoint 서비스는 향상 된 성과 기록표, 분해 트리 및 KPI 세부 정보 보고서와 더 많은 차트 종류, 더 나은 시간 인텔리전스 필터링 기능 및 향상 된 내게 필요한 옵션 규정 준수와 같은 새 시각화를 제공합니다. 자세한 내용은 [PerformancePoint Services (SharePoint Server 2010)에 대 한 새로운](https://go.microsoft.com/fwlink/?linkid=841343)를 참조 합니다.
  
## <a name="where-can-i-get-help-with-my-upgrade"></a>내 업그레이드 도움말을 어디서 구할 수 있습니까?

업그레이드 하는 온-프레미스 또는 Office 365로 이동 중일, 여부 하려는 Microsoft 파트너와 협력 하는 것이 좋습니다. 비즈니스 요구 사항 및 배포와 함께 도우미에 가장 잘 맞는 솔루션을 식별 된 파트너 수 있습니다. [Microsoft 파트너 센터](https://go.microsoft.com/fwlink/?linkid=841249)방문 하 고 검색 필터를 사용 하 여 솔루션 공급자를 찾을 수 있습니다.
