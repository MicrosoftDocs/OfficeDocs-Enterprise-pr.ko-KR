# <a name="move-in-ready"></a>이동-준비 완료

 **요약:** Microsoft의 클라우드 서비스에 기본 제공 되는 저장소 옵션을 클라우드 집합에 정보를 가져옵니다.
  
이동-에서 "준비" 저장소 솔루션:
  
- 미리 패키지 된 솔루션을 즉시 사용
    
- 준비가 되 면 이동 하 고, 기존 서비스와 함께 제공
    
- 제한 된 사용자 지정을 제공 합니다.
    
다음 섹션에서는 각 "이동-준비" 저장소의 세부 정보에 설명 옵션입니다.
  
## <a name="sharepoint-online-in-office-365"></a>Office 365의에서 SharePoint Online

### <a name="features"></a>기능

- 파일은 안전 하 고 신뢰할 수 있는, 액세스할 수 있는 여러 장치에서 수집 하 고 어디에서 든 지
    
- 콘텐츠를 검색을 통해 쉽게 검색할 수
    
- 워크플로 및 보존 정책을 통해 간편 하 게 사용 하 고 콘텐츠를 유지 관리
    
- 웹 api (영문) 또는 브라우저를 통해 액세스 또는 오프 라인 콘텐츠 될 수 있습니다.
    
### <a name="common-uses"></a>일반적으로 사용

- 회사 또는 조직 포털
    
- 팀 기반 공동 작업
    
- 엔터프라이즈 검색
    
### <a name="key-storage-scenarios"></a>키 저장소 시나리오

- 공동 작업
    
- 회사 통신
    
- 비디오를 관리 합니다.
    
### <a name="resources"></a>리소스

자세한 내용은 [여기](https://support.office.com/article/SharePoint-Online-and-OneDrive-for-Business-software-boundaries-and-limits-8f34ff47-b749-408b-abc0-b605e1f6d498)합니다.
  
비용 정보에 대 한 클릭 [여기](https://products.office.com/business/compare-office-365-for-business-plans)합니다.

## <a name="azure-backup"></a>Azure 백업

### <a name="features"></a>기능

- Windows Server 및 Windows Server Essentials 로부터 받은 데이터를 보호 하는 원활 하 게
    
- System Center Data Protection Manager를 포함합니다.
    
- 데이터는 두 Azure 데이터 센터 데이터의 6 개의 복사본을 유지 하는 지리적으로 분산 복제 저장소에 저장 된
    
- 델타 변경, 정의 된 주파수, 압축, 암호화, 긴 보존 및 대역폭 제한 지원
    
### <a name="common-uses"></a>일반적으로 사용

- 다양 한 온-프레미스 (SharePoint, SQL, Exchange)와 Azure (Vm) 원본에서에서 데이터를 보호 합니다.
    
### <a name="key-storage-scenarios"></a>키 저장소 시나리오

- 백업 파일
