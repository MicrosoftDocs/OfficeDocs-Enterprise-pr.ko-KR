### <a name="servicing-resource-requests-between-servers"></a>서버간 리소스 요청에 응답

서버-서버 인증은 Exchange Server 2013, Lync Server 2013 및 사용자를 대신 하 여 다른 서버의 요청 리소스에는 서버를 허용 하는 SharePoint Server 2013의 새로운 기능입니다. 이 기능은 사용 업계 표준 OAuth (Open Authorization) 2.0 프로토콜입니다. 서버-서버 인증을 사용 하면 사이트 사서함 eDiscovery, 고해상도 사용자 사진 등의 많은 새로운 시나리오입니다. 
  
 **서버 제품**

### <a name="server-to-server-trust-relationships"></a>서버 간 트러스트 관계

들어오는 리소스 요청을 처리 하는 서버에 대 한 요청을 만드는 서버를 신뢰 해야 것입니다. 이 트러스트를 설정 하기 위해 서버 간 트러스트 관계를 구성 해야 합니다. 
  
서버 간 트러스트 관계가 한 가지 방법입니다. Exchange 서버에서 SharePoint Server 트러스트 리소스 요청을 실행 하는 서버를 포함 하지만 Exchange 서버는 리소스 요청을 실행 하는 서버에서 신뢰 하지 않으므로 Exchange 2013 서버를 신뢰 하도록 SharePoint 2013을 실행 하는 서버를 구성 하는 경우 SharePoint 서버입니다. 원활 하 게 통합에 대 한 양방향 트러스트를 설정 해야 합니다. 
  
함께 제공 된 다이어그램 양방향 트러스트로 설정 하는 서버 간 트러스트 관계를 보여줍니다. 표시 된 양방향 트러스트 관계는 Exchange Server, SharePoint 서버 및 Lync 서버 사이 있습니다. 각 서버 유형에 양방향 트러스트를 각각의 다른 두 서버와 는합니다. 
  
### <a name="configuration"></a>구성

서버간 인증 트러스트를 구성 하려면 사용자를 대신 하 여 리소스 요청을 전송 하는 각 서버에 해당 하는 새 신뢰할 수 있는 보안 토큰 발급자를 추가 해야 합니다. 각 유형의 서버에는 구성 정보 및 액세스 토큰 서명 인증서의 공용 부분을 포함 하는 표기법 JSON (JavaScript Object) 메타 데이터 끝점입니다. 서버간 인증 트러스트를 구성 하는 부분이 다른 서버의 JSON 메타 데이터 끝점을 지정 합니다. 
  
다음 표에서 각 서버에 대 한 JSON 메타 데이터 끝점을 보여줍니다. 다음 표에서가 보여줍니다. 
  
- Https:// JSON 메타 데이터 끝점을 사용 하 여 Exchange 서버<server name>/autodiscover/metadata/json/1 
    
- Lync 서버 https:// JSON 메타 데이터 끝점을와<server name>/metadata/json/1 
    
- JSON 메타 데이터 끝점을 https://와 SharePoint 서버<web app name>/_layouts/15/metadata/json/1 
    
### <a name="example-how-server-to-server-authentication-works-for-ediscovery-between-sharepoint-and-exchange"></a>예: SharePoint와 Exchange 간의 eDiscovery에 대 한 방식으로 서버-서버 인증 작동

이 예제에서는 Exchange 2013 서버 서버 간 트러스트를 사용 하 여 SharePoint Server를 실행 하는 서버를 신뢰 하도록 구성 된 했습니다. SharePoint Server를 실행 하는 서버에서 eDiscovery 센터가 Exchange 서버에 있는 사서함에 데이터를 포함 하도록 구성 합니다. 
  
다른 서버에 있는 리소스에 대 한 요청을 사용 하면 대상 서버에서 웹 서버 서비스에 전송 되는 액세스 토큰의 형태로 합니다. 
  
함께 제공 되는 다이어그램에서는 쿼리 및 액세스 토큰에 두 대의 서버 간에 이동 하는 방법을 보여줍니다. 
  
1. EDiscovery 관리자가 Exchange 서버에 있는 리소스를 포함 하는 SharePoint Server를 실행 하는 서버에 쿼리를 보냅니다. 
    
2. SharePoint Server를 실행 하는 서버에는 사용자 및 요청한 리소스를 식별 하는 액세스 토큰을 생성 합니다. 
    
3. SharePoint Server를 실행 하는 서버에서 Exchange 서버에 대 한 액세스 토큰을 보냅니다. 
    
4. Exchange 서버는 액세스 토큰의 유효성을 검사 하 고 쿼리 결과 보냅니다. 
    
5. SharePoint Server를 실행 하는 서버는 eDiscovery 쿼리 결과 eDiscovery 관리자의 컴퓨터에 보냅니다.

## <a name="unified-contact-store-tab"></a>통합된 연락처 저장소 탭

### <a name="exchange-2013-is-the-contact-store-for-all-office-applications"></a>Exchange 2013은 모든 Office 응용 프로그램에 대 한 연락처 저장소

통합된 연락처 저장소 (UCS) Microsoft Office 제품에서 일관 된 연락처 경험을 제공합니다. 사용자가 자신의 Exchange 2013 사서함에 있는 모든 연락처 정보를 저장합니다. 같은 연락처 정보를 사용할 수 있는 전역으로 Lync, Exchange, Outlook 및 Outlook Web App. 
  
 **서버 제품**
  
- Exchange Server 2013 
    
- Lync Server 2013 
    
 **구성**
  
- Exchange Server 2013과 Lync Server 2013 간에 서버간-인증을 구성 합니다. 
    
- Lync 2013의 통합된 연락처 저장소 정책을 사용 하도록 설정 (기본 설정) 합니다. 
    
예 구성 스크립트 Jens Trier Rasmussen의 블로그 (영문) 문서를 참조 합니다. 
  
- Exchange 2013과 Lync Server 2013 (http://aka.ms/Oyg7fh) 통합 
    
 **작동 방식**

Lync 현재 상태 정보를 사용 하 여 Lync, Outlook 및 SharePoint 사이에서 현재 상태 정보의 일관성 있는 보기를 가질 수 있습니다. Outlook Outlook와 동일한 컴퓨터에 로컬로 설치 되는 Lync에서 직접 현재 상태 정보를 쿼리 합니다. 사용자가 SharePoint 서버에서 현재 상태 정보를 볼 때 현재 상태 정보는 로컬 컴퓨터에서 Lync 하 여 쿼리 됩니다.

2. 사용자가 로그인을 자신의 Exchange 2013 사서함 및 Exchange CAS 서버를 쿼리 합니다 Lync 현재 상태 정보에 대 한 sip / MTLS:5061 합니다. 
    
자세한 내용은 [Microsoft Lync Server 2013 통합 (영문) 및 Microsoft Outlook Web App 2013을](https://go.microsoft.com/fwlink/?LinkId=313522)참조 하십시오.

1. 사용자가 로그인 Lync 2013에 sip / Lync server를 호출 하는 TLS:5061 합니다. 
    
2. A. 사용자가 Office 2013에서 Outlook 통해 Exchange 2013 사서함에 로그인 합니다. HTTPS/443을 통한 Exchange 클라이언트 액세스 서버 (CA)를 액세스 하는 클라이언트 컴퓨터 및 Exchange의 내부 통신으로 Exchange 사서함 서버도 호출 하는 것입니다. 
    
3. A. outlook 호출 Lync 현재 상태 정보를 검색 하는 Outlook와 동일한 컴퓨터에 설치 합니다. 
    
4. B. 사용자는 HTTP 또는 SharePoint 서버를 호출 하는 HTTPS를 통해 SharePoint 내 사이트에 연결 합니다. 
    
5. B. Internet Explorer Lync를 브라우저와 동일한 컴퓨터에 설치 되는 현재 상태 정보를 검색할 수를 호출 합니다.

2. 통화는 수신자의 Lync 홈 서버로 라우팅됩니다. 
    
3. Lync Server Lync 2013에서 수신자의 활성 끝점에서 신호음이 울립니다. 
    
4. 통화에 응답 하지 Exchange CAS (호출 라우터)에 대 한 호출 음성 메일 (Exchange UM)로 라우팅됩니다. 
    
Lync 2013을 사용 하 여 통화를 시작 하는 발신자: 
  
1. B에 전화를 걸 PSTN을 사용 하 여 수신자의 전화번호입니다. 
    
2. Lync 서버 중재 서버에 IP 게이트웨이에서 PSTN 호출이 라우팅됩니다. 
    
3. 중재 서버 routs 수신자의 Lync 홈 서버에 대 한 호출입니다. 
    
4. Lync Server Lync 2013에서 수신자의 활성 끝점에서 신호음이 울립니다. 
    
5. 통화에 응답 하지 Exchange CAS (호출 라우터)에 대 한 호출 음성 메일 (Exchange UM)로 라우팅됩니다. 
    
## <a name="meeting-recordings-tab"></a>모임 녹음/녹화 탭

### <a name="publish-your-meeting-recordings-on-your-sharepoint-team-site"></a>SharePoint 팀 사이트에 모임 녹음/녹화를 게시 합니다.

모임 녹음/녹화는 통합된 커뮤니케이션의 핵심 구성 요소입니다. 모임 녹음/녹화를 공유 하려면 팀 사이트에서 SharePoint 자산 라이브러리를 사용 하 여 모임 녹음/녹화를 저장 하는 것이 좋습니다. 
  
클라이언트 제품: 
  
- Lync 2013 
    
서버 제품: 
  
- 서버 제품: 
    
- SharePoint 2013 
    
선행 조건: 
  
- Lync 2013-모임 녹음/녹화는 Lync 2013의 클라이언트쪽 기능. 
    
- SharePoint 2013-모임 녹음/녹화를 이미 실행 되 고 위쪽 저장 하려는 팀 사이트를 해야 합니다. 
    
 **기록 되?**
  
다음은 MP4 파일에 하는 동안 기록 (각 글머리 기호의 녹음/녹화의 형식을 나타내는 아이콘과 함께 앞에) 회의: 
  
- 모든 오디오 
    
- 활성 스피커의 (사용) 하는 경우 비디오 
    
- 파노라마 비디오 (사용 된 경우) 
    
- 제공 되는 모든 콘텐츠 
    
- 인스턴트 메시지 * 
    
* 모임 내에서 인스턴트 메시지에만 포함 됩니다. 모임 참가자 간의 피어-투-피어 메시징 모임에 속하지 않는 및 따라서 전화 캡처되지 않습니다. 
  
포스터에는 서로 다른 두 시나리오에 대 한 두 다이어그램 포함 됩니다. 
  
- 모임 녹음/녹화를 게시 하기 위한 준비 
    
- 녹음/녹화 하 고 Lync 클라이언트를 사용 하 여 모임 게시 
    
### <a name="preparing-for-publishing-meeting-recordings"></a>모임 녹음/녹화를 게시 하기 위한 준비

다이어그램에서는 팀 사이트, 중앙 관리 센터 및 인터넷 정보 서비스 (IIS) 서버와 SharePoint Server 2013을 보여줍니다. 
  
팀 사이트에는 다음이 포함 됩니다. 
  
- 자산 라이브러리 응용 프로그램입니다. 
    
- 팀 구성원 보낼 모임 녹음/녹화 모임 자산 라이브러리입니다. 
    
중앙 관리 센터에는 웹 응용 프로그램 일반 설정을 포함합니다. 
  
IIS 서버에 IIS 설정을 포함합니다. 
  
모임 녹음/녹화를 게시 하기 위한 준비 합니다. 
  
1. SharePoint 팀 사이트에서 업로드 크기 제한 또는 연결 시간 제한으로 인해 모임 녹음/녹화, 2 및 3 추가 단계를 수행 하는 작업을 수 없는 경우 자산 라이브러리 app.를 필요에 따라 추가 합니다. 
    
2. SharePoint 중앙 관리에서 팀 사이트 모음을 포함 하는 웹 응용 프로그램에 대 한 최대 업로드 크기 설정을 변경 합니다. 
    
3. IIS 서버 설정에서 팀 사이트 모음을 포함 하는 웹사이트에 대 한 IIS 연결 제한 시간을 늘립니다. 
    
 **디지털 자산 라이브러리**
  
디지털 자산 라이브러리는 특정 용량 및 성능 영향을 주는 비디오를 포함 하는 자산 라이브러리입니다. 자세한 내용은 http://aka.ms/O1vq5w에 있는 SharePoint Server 2013의 디지털 자산 라이브러리 계획을 참조 하십시오. 포스터에는이 정보에 액세스 하기 위해 QR 코드를 포함 됩니다. 
  
### <a name="recording-and-publishing-a-meeting-using-the-lync-client"></a>녹음/녹화 하 고 Lync 클라이언트를 사용 하 여 모임 게시

다이어그램은 Lync를 사용 하 여 모임에 참가 하는 사용자를 보여줍니다. 모임의 모임 콘텐츠 인 한 MP4 파일을 만드는 Lync 클라이언트를 사용 하 여 기록 됩니다. MP4 녹음/녹화는 사용자의 컴퓨터에 Lync 기록 폴더에 저장 됩니다. MP4 녹음/녹화를 삽입할 수 있습니다를 위 키, SharePoint 페이지 또는 블로그 (영문)로 하 여 모임 자산 라이브러리를 이동할 수 있습니다. 
  
 **기록 하 고 Lync 클라이언트를 사용 하 여 모임 게시**
  
1. Lync 클라이언트를 사용 하 여 모임 녹음/녹화를 시작 합니다. 
    
2. 모임 콘텐츠는 모임 중 MP4 파일에 기록 됩니다. 
    
3. 모임 살펴보며 MP4 녹음/녹화 사용자의 컴퓨터에서 녹음/녹화 폴더에 나타납니다 (c:\\사용자\\<username>\\비디오\\Lync 녹음/녹화). 필요에 따라 Lync 클라이언트와 함께 설치 하는 Lync Recording Manager app를 사용 하 여 모임 녹음/녹화를 사용자 지정할 수 있습니다. 
    
4. SharePoint 자산 라이브러리에 기록 하 고 회의 끌어서 설정 합니다. 
    
5. 선택 사항: 녹음/녹화가 자산 라이브러리에 참가 삽입할 수 있습니다 모든 SharePoint 페이지에 있습니다. 이 단계에 대 한 자세한 내용은 http://aka.ms/R61q35에 있는 만들기 및 SharePoint와 Lync 온라인 교육 비디오 게시 Office 365 블로그 항목을 참조 하십시오. 
    
 **비디오 축소판 그림**
  
비디오 축소판 그림 자산 라이브러리의 모양과 느낌을 향상 시킵니다. 모임 녹음/녹화에 대 한 축소판 그림을 작성 하는 방법에 대 한 자세한 내용은, 캡처를 참조 하거나 비디오 축소판 그림을를 변경 하려면 http://aka.ms/Kupj85에 배치 합니다. 포스터에는이 정보에 액세스 하기 위해 QR 코드를 포함 됩니다.
