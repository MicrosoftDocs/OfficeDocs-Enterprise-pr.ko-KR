# <a name="prepare-directory-attributes-for-synchronization-with-office-365-by-using-the-idfix-tool"></a>IdFix 도구를 사용하여 Office 365와의 동기화를 위한 디렉터리 특성 준비
이 항목에서는 IdFix 도구를 실행 하는 방법에 대 한 자세한 지침을 소개 일반적인 오류 발생할 수 픽스, 예제 및 오류 수가 많은 경우 취해야 할 조치에 대 한 모범 사례를 제시 합니다.
  
## <a name="fixing-errors-in-your-directory-by-using-the-idfix-gui"></a>IdFix GUI를 사용하여 디렉터리의 오류 수정
디렉터리의 문제를 검색 하 고이 항목의 설명에 따라 다음 GUI에서 오류를 수정 하려면 [Office 365 IdFix 도구를 실행](install-and-run-idfix.md) 합니다. 빈 표 도구에 의해 반환 되는 경우에 오류가 발견 된 합니다. 디렉터리의 문제 많은 경우이 도구는 오류를 반환 하는 경우 번거로운 수 있습니다. 이 해결 하는 한 가지 방법은 먼저 한 가지 유형의 모든 오류를 수정한 다음 다음 형식으로 이동 하는 합니다. 
  
1. 변경을 시작하기 전에 IdFix에서 제공되는 권장 지침을 확인합니다.
    
2. IdFix가 반환한 오류 목록을 검토합니다. 오류 유형을 표시하는 열 맨 위에 있는 **ERROR**를 클릭하여 오류 유형별로 오류를 정렬할 수 있습니다. 둘 이상의 오류가 단일 특성과 관련되어 있으면 오류가 하나의 행으로 결합됩니다. 가능한 경우 IdFix에서는 **UPDATE** 열에 권장 수정 방법이 제공됩니다. 수정 방법은 개체와 연결된 다른 특성을 확인한 결과를 기반으로 합니다. 이러한 수정 내용이 기존 디렉터리 내용보다 낫더라도 어떤 값이 정확한지는 사용자만이 결정할 수 있습니다. 
    
3. IdFix 중복 오류에 대 한 픽스에 대 한 제안 있으면 수정 프로그램에 의해 식별 된 **업데이트** 열에서 값의 시작 부분에 세개의 플래그 중 하나를 등 `[E]john.doe@contoso.com`합니다. 변경 내용을 적용 하는 경우의 추천 단어를 수락 하면 플래그 디렉터리에 삽입 되지 않습니다. 추천 단어에 플래그를 적용 되며 등을 다음 값만 `john.doe@contoso.com`합니다. 추천 단어에 적용 하려는 경우 **작업** 열에서 일치 하는 작업을 선택 합니다. 플래그는 다음과 같이 동작을 나타냅니다. 
    
 - **[C]** 제안 동작 **COMPLETE**. 값을 편집할 필요가 없습니다.
    
 - **[E]** 제안 동작 **EDIT**. 디렉터리의 다른 값과 충돌하지 않게 하려면 값을 변경해야 합니다.
    
 - **[R]** 제안 동작 **REMOVE**. 값은 메일 사용이 가능하지 않은 개체의 SMTP 프록시이므로 제거해도 됩니다.
    
4. 한번 했을 때를 잘 읽고 이해 하는 오류를, 변경 내용으로 **업데이트** 열에서 항목을 업데이트 한 다음 **동작** 에 열 선택 변경을 구현 하기 위해 수행 하는 IdFix 사용할 합니다. 두 사용자가 발생할 수 있는 예는 `proxyAddress` 중복 것으로 식별 합니다. 한 명의 사용자만 사용할 수는 `proxyAddress` 메일 배달을 위한 합니다. 이 문제를 해결 하려면 올바른 값을 가진 사용자에 대 한 **작업** 열 **전체** 를 표시 하 고 다른 사용자에 대 한 **제거** **작업** 열을 표시 합니다. 이렇게 하면 제거는 `proxyAddress` 적용할 사용자에서 특성이 `proxyAddress` 속하지 사용자에 게 변경 되지 않음 사람 해지고이 `proxyAddress` 정확한 합니다.

## <a name="operations-you-can-perform-by-using-idfix"></a>IdFix를 사용하여 수행할 수 있는 작업
오류를 해결 하려면 **작업** 드롭다운 목록에서 옵션을 선택 합니다. 다음 표에서 IdFix 도구를 사용 하 여 특성에서 수행할 수 있는 **작업** 작업을 설명 합니다. **작업** 열을 비워 IdFix 도구는 디렉터리에서 해당 특정 오류에 대 조치를 취할 하지 않습니다. 

|**작업**|**작업 설명**|**예제**|
|:-----|:-----|:-----|
|**COMPLETE** | 원래 값을 사용할 수 있으며 오류로 식별되었더라도 변경하지 않는 것이 좋습니다. | 사용자 두 명의 proxyAddress가 중복된 것으로 확인되었습니다. 한 명만 이 주소 값을 사용하여 메일을 받을 수 있습니다. 올바른 값을 갖는 사용자를 **COMPLETE**로 표시합니다. |
|**제거** | 특성 값은 원본 개체에서 삭제 됩니다. 다중값된 특성의 경우 `proxyAddresses`, 표시 되는 개별 값만 삭제 됩니다. | 사용자 두 명의 proxyAddress가 중복된 것으로 확인되었습니다. 한 명만 이 주소 값을 사용하여 메일을 받을 수 있습니다. 중복된 값을 갖는 사용자를 **REMOVE**로 표시합니다. |
|**편집** | **업데이트** 열에 대 한 정보를 특성 값을 수정 하려면 사용 됩니다. 유효한 **업데이트** 값 IdFix에서 제안 된 했습니다, 하는 경우 다음 **작업** 열에서 **편집** 을 선택 하 고 오류를 이동 합니다. 추천 단어에 마음에 들지 않으면 **업데이트** 열에서 새 암호를 입력 하 고 **작업** 열에서 **편집**을 선택 합니다. ||
|**실행 취소** | 이 옵션은 트랜잭션 로그에서 복원한 경우에만 사용할 수 있습니다. **UNDO**를 선택하면 특성 값이 원래 값으로 복원됩니다. ||
|**FAIL** | **업데이트** 값에 AD DS 규칙에 알 수 없는 충돌 하는 경우에이 값이 반환 됩니다. 이 경우 편집할 수 있습니다 **업데이트** 열에서 값 다시를 알고 있는 경우 어떤 오류가 발생 합니다. ADSI 편집을 사용 하 여 개체의 값을 분석할 수 있습니다. 자세한 내용은 [ADSI 편집 (adsiedit.msc)을](https://go.microsoft.com/fwlink/p/?LinkId=401170)참조 하십시오. ||

특정 오류 또는 일괄 오류에 대해 **ACTION**을 선택한 경우 **적용**을 클릭합니다. **적용**을 클릭하면 디렉터리가 변경됩니다. 여러 오류에 대한 수정 내용을 제공한 다음 **적용**을 클릭하면 IdFix가 오류를 모두 한 번에 변경합니다.

IdFix 수행한 픽스 새 오류를 도입 하지 않았으므로 확인을 다시 실행 합니다. 이러한 단계에 필요한 횟수 만큼 반복할 수 있습니다. 통과 하도록 하는 프로세스는 몇 번 동기화 하기 전에 것이 좋습니다.
    
## <a name="changing-the-rule-set-used-by-idfix"></a>IdFix에서 사용되는 규칙 집합 변경
기본적으로 IdFix 디렉터리에 있는 항목을 테스트로 설정 하는 다중 테 넌 트 규칙을 사용 합니다. 이것은 대부분 Office 365에 대해 설정 오른쪽 규칙 고객 = 합니다. 그러나 Office 365 전용 또는 ITAR (팔 규정에 국제 트래픽) 고객 인 경우 대신 설정 하는 전용된 규칙을 사용 하 여 IdFix를 구성할 수 있습니다. 확실치 않은 경우 어떤 유형의 하는 고객을 안전 하 게이 단계를 건너뛸 수 있습니다. 전용으로 설정 하는 규칙을 설정 하는 메뉴 모음에서 기어 아이콘을 클릭 하 고 **전용**을 클릭 합니다.
  
## <a name="changing-the-scope-of-the-search-used-by-idfix"></a>IdFix에서 사용되는 검색 범위 변경
기본적으로 IdFix는 전체 디렉터리를 검색합니다. 원할 경우 특정 하위 트리를 대신 검색하도록 도구를 구성할 수 있습니다. 이렇게 하려면 메뉴 모음에서 필터 아이콘을 클릭하고 올바른 하위 트리를 입력합니다.
  
## <a name="rolling-back-your-changes-by-using-the-idfix-gui"></a>IdFix GUI를 사용하여 변경 내용 롤백
변경 내용을 적용 하려면 **적용** 클릭할 때마다 IdFix 도구 라는 방금 변경한을 나열 하는 트랜잭션 로그를 별도 파일을 만듭니다. 트랜잭션 로그를 사용 하 여 실수 하는 경우 가장 최근 로그에는 해당 변경 내용을 취소할 수 수 있습니다. 업데이트 하는 동안 실수 하는 경우에 **실행 취소**를 클릭 하 여 가장 최근에 적용된 된 변경 내용을 취소할 수 있습니다. **실행 취소**를 클릭 하면 IdFix 트랜잭션 로그를 사용 하 여 가장 최근의 트랜잭션 로그에 있는 변경 내용만 롤백합니다. 트랜잭션 로그를 사용 하는 방법에 대 한 자세한 내용은 참조 [참조: Office 365 IdFix 트랜잭션 로그](idfix-transaction-log.md)합니다.
